O Problema Introdução

Por que IAs erram em projetos grandes?

  • Perda de Contexto: Muita informação, pouca relevância.
  • Padrões Genéricos: IA aplica a "média da internet".
  • Deriva de Contexto: "Esquecimento" em sessões longas.
  • Ausência de Guardrails: IA busca o caminho mais rápido, não o correto.

A Analogia do GPS

A IA é o GPS.

O CLAUDE.md é o Manual do Carro:

  • Evite pedágios específicos.
  • A 2ª marcha engasga.
  • Manobra específica na garagem.

Transforma um navegador genérico em um copiloto do SEU projeto.

O Padrão WHY-WHAT-HOW

Seção Pergunta Efeito
WHY Para que serve? Delimita o escopo.
WHAT Como está organizado? Dá visão espacial (árvore).
HOW Como eu opero? Comandos sem invenção.

Exemplo Real (Root)

## WHY - Purpose
PX-Nexus: Monorepo full-stack com tRPC.

## WHAT - Architecture
├── apps/api  # NestJS
└── apps/web  # React 19

## HOW - Commands
pnpm nexus    # Dev mode
pnpm db:migrate -- --name X # OBRIGATÓRIO

Documentação em Camadas

Carregue apenas o que é necessário.

Nível 1: Root

Global: Stack, Comandos Principais.

Nível 2: Local

Domínio: Regras de API, Web ou DB.

Nível 3: Agent Docs

Doutrina: Arquitetura profunda e Step-by-Step.

Economia de Tokens

Trabalhando no Frontend?

✅ Carrega Root + Web CLAUDE.md

❌ Ignora DB Migrations CLAUDE.md

Resulta em respostas mais rápidas e baratas.

Guardrails Instruções Negativas

IAs são otimistas e buscam o caminho mais fácil.

Use o formato:
Proibição + Alternativa + Justificativa

Formatação de Peso

  • Negrito: Ênfase semântica.
  • Maiúsculo: Instrução forte.
  • Bilinguismo: Use Português para "Regras da Casa".

Exemplo Crítico

## REGRA CRÍTICA: MIGRATIONS
**PROIBIDO:** `db:push` - Nunca usar!
**OBRIGATÓRIO:** Toda alteração via migration.

**Por que?**
- Histórico versionado.
- Reproduzível em prod.

Spec-First / CDD

Checkpoint-Driven Development

Diferencie O que o sistema faz (Código) de O que deveria fazer (Spec).

Ciclo de Vida da Spec

  1. Discovery: Escreva a intenção.
  2. Validation: IA revisa o contrato.
  3. Execution: Implemente com checkpoints.
  4. Pruning: Arquive para memória histórica.

Bilinguismo Estratégico

Inglês

Estrutura, Stack, Termos Técnicos.

(A IA é treinada nisso)

Português

Guardrails, Regras de Negócio, Justificativas.

(Contraste visual = Regra Local)

Agent Docs Doutrina

"Se eu não explicar isso, a IA vai adivinhar certo?"

Se a resposta for NÃO, crie um Agent Doc.

O que documentar?

  • Arquitetura: Por que tRPC e não REST?
  • Fluxo: Como o dado viaja do DB à UI?
  • Runbooks: Passo-a-passo para criar uma feature.

Session Handover

O session_ledger.md é a memória de curto prazo entre chats.

# Session Ledger
## O que foi feito:
- Criado schema de audit_logs.
## O que falta:
- Criar página de visualização.
## Decisões:
- Usamos soft delete por segurança.

SYNC ALERT

"Se alterou código, verifique se a documentação precisa de atualização."

Torne isso um passo obrigatório no CLAUDE.md.

Checklist de Maturidade

  • 🟣 Nível 3: Specs ativas e Session Ledger.
  • 🔥 Nível 4: Doutrina (Agent Docs) e Onboarding Autônomo.
