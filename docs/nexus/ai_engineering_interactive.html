Por que IAs erram?

Agentes de IA são como GPS de alta precisão: eles sabem navegar por qualquer lugar, mas não conhecem os atalhos, buracos ou regras específicas da sua vizinhança.

🧠
Perda de Contexto

Em projetos grandes, a IA lê arquivos aleatórios e consome tokens sem entender a hierarquia.

🌐
Padrões Genéricos

Sem guia, ela aplica o "padrão da internet" que pode ser oposto ao do seu projeto.

O que é Engenharia de Contexto?

É a prática de estruturar informação para que agentes de IA consumam o contexto certo, no momento certo, com a profundidade certa.

O Padrão WHY-WHAT-HOW

Todo documento de contexto deve responder a estas 3 perguntas em ordem sequencial.

WHY

Para que este módulo existe?

Delimita o escopo e evita sugestões fora do domínio.

WHAT

Como está organizado?

Dá visão espacial. A IA localiza arquivos sem varrer o disco.

HOW

Quais comandos usar?

Instruções operacionais. A IA executa sem inventar.

## WHY - Project Purpose
PX-Nexus connects frontend React to NestJS via tRPC.

## WHAT - Architecture
px-nexus/
├── apps/api  # NestJS Server
└── apps/web  # React Frontend

## HOW - Key Commands
pnpm nexus      # Start dev
pnpm db:migrate # Create migration

1. Documentação em Camadas

Context Layering distribui o conhecimento em níveis hierárquicos. Carregue apenas o necessário.

Nível 1: Root

Visão macro. Sempre carregado.

Nível 2: App-Specific

Regras locais. Carregado por proximidade.

Nível 3: Doutrina

Agent Docs e Specs. Carregado sob demanda.

Dica: Se você trabalha em apps/web/, a IA não precisa ler as regras de migration do banco de dados.

2. Guardrails (Instruções Negativas)

A IA é otimista. Guardrails criam limites explícitos e destacam o que NÃO fazer.

PROIBIDO: db:push - Nunca usar para alterações de schema!
OBRIGATÓRIO: Toda alteração DEVE ser via migration: pnpm db:migrate

Como calibrar

Específico > Genérico. Use negrito, maiúsculas e contraste bilingue.

3. Spec-First (CDD)

Checkpoint-Driven Development: Use especificações como "âncoras de verdade" para evitar que a IA se perca em sessões longas.

Fluxo CDD:

  1. Discovery: Escreva a spec em .claude/specs/
  2. Validation: Revise o contrato com a IA.
  3. Execution: Implemente com a spec como guia.
  4. Pruning: Mova para archive/ após concluir.

4. Bilinguismo Estratégico

O contraste entre idiomas sinaliza importância para o modelo.

Inglês 🇺🇸

Estrutura, stack e comandos técnicos. A IA tem mais dados de treino.

Português 🇧🇷

Guardrails, justificativas e "regras da casa". Cria contraste visual.

5. Agent Docs (Doutrina)

Quando o código não é autoexplicativo, você precisa de Agent Docs.

Pergunte-se: "Se eu não explicar isso, a IA vai adivinhar certo?"

  • Decisões Arquiteturais (O "Porquê")
  • Fluxos de Dados Complexos
  • Workflows Step-by-Step

6. Session Handover

Sessões de IA expiram ou perdem contexto. O session_ledger.md é sua memória externa.

## Decisões: Soft delete usado para segurança
